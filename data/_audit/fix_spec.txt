## Root cause (verified by reading the code)

`_scheme_score` (lines 835-844) rewards length and the presence of the words `fund`/`index`:

```python

    + (1.0 if any(t in s2 for t in ("scheme","plan","fof","etf","index")) else 0.0) \
    + 0.1 * min(len(s.split()), 6)
```

The `+0.1 * word_count` term (capped at 6 words) means a verbose objective sentence like *"360 ONE Balanced Hybrid Fund (An open ended scheme investing in...)"* or *"The Scheme seeks to generate long-term capital appreciation..."* outscores the short real name *"360 ONE Balanced Hybrid Fund"* ŌĆö both contain `fund`, but the objective is always longer, so it always wins the word-count tiebreak. _BULLET_LEAD_RE = re.compile(r"^[\*\u2022\u25cf\u25aa\u2219\u00b7\-ŌĆōŌĆö#┬¦]+\s")

```python
# Anchored on a separator (-, ŌĆō, ŌĆö, :, ,) followed by an open/close-ended (or interval-fund) clause,
# consuming to end of string. Used only to RECOVER the real head; never to reject.

    r"\s*[\-ŌĆōŌĆö:,]\s*(?:an?\s+)?(?:open[\s-]?ended|close[d]?[\s-]?ended|interval)\b.*$",
    re.I,
)


def _looks_like_objective(s: str) -> bool:
    """True if `s` reads like an investment objective / SEBI disclaimer / riskometer / bullet /
    bare AMC house-name ŌĆö i.e. NOT a real scheme name. Conservative: a real name that merely
    contains 'Fund'/'ETF'/'Index'/'FOF'/'Plan' or an '(Erstwhile/Formerly ŌĆ”)' alias is NOT flagged
    unless it ALSO carries one of the objective phrases above."""
    if not s:
        return False
    low = s.lower().strip()
    if _BULLET_LEAD_RE.match(s):                      # leading ŌĆó/*/- footnote marker
        return True
    if _HOUSE_ONLY_RE.match(low):                     # bare "<House> Mutual Fund"
        return True
    if any(p in low for p in _OBJECTIVE_PHRASES):     # any objective/disclaimer phrase
        return True
    # A very long, sentence-like cell with terminal punctuation and no obvious scheme token is prose.
    if len(s.split()) >= 14 and (low.endswith(".") or "," in low) \
            and not any(t in low for t in (" fund", " etf", " fof", " plan", " index", " scheme")):
        return True
    return False


def _strip_objective_tail(s: str) -> str:
    """Recover the real head of a 'name_plus_objective' cell:
    'X Balanced Hybrid Fund - An open ended scheme investing in ŌĆ”' -> 'X Balanced Hybrid Fund'.
    Only strips an objective tail introduced by a separator; leaves clean names untouched and
    NEVER touches an '(Erstwhile/Formerly ŌĆ”)' alias (those use parentheses, not this separator)."""
    if not s:
        return s
    head = _OBJECTIVE_TAIL_RE.sub("", s).strip()
    # Guard: don't strip down to nothing or to a fragment that lost its scheme token.
    if head and len(head) >= 4:
        return head
    return s
```

## Why this is conservative (does not reject real names)

- `_looks_like_objective` fires **only** on (a) a leading bullet/asterisk, (b) a *tail-anchored* bare `"<House> Mutual Fund"`, (c) an explicit objective/disclaimer **phrase**, or (d) a 14+-word sentence with terminal punctuation **and** no scheme token. None of those match `"360 ONE QUANT FUND"`, `"DSP Equity FOF"`, `"Bharat Bond ETF - April 2030"`, `"Edelweiss Nifty 50 Index Fund"`, or an `"(Erstwhile/Formerly ŌĆ”)"` alias.
- The `(d)` prose heuristic explicitly **exempts** any cell containing ` fund`/` etf`/` fof`/` plan`/` index`/` scheme`, so a long but legitimate name (e.g. *"Edelweiss Nifty LargeMidcap 250 Plus 8-13 Yr G-Sec 70:30 Index Fund"*, ~10 words and no terminal period anyway) is never flagged.
- `_strip_objective_tail` only cuts at a separator (`-`/`ŌĆō`/`ŌĆö`/`:`/`,`) **followed by** an `open/close-ended/interval` clause. The `"(Erstwhile ŌĆ”)"` aliases use parentheses (handled by the existing `_clean`), so this never touches them. `"Bharat Bond ETF - April 2030"` is safe: after the `-` comes `April`, not an open/close-ended clause, so the regex doesn't match.
- The 147 garbage cases map cleanly: the `name_plus_objective` rows (idx 0-10, 263, 271-272 ŌĆ”) are recovered by Part 3's tail-strip; the pure `objective` rows (idx 100-120, real_name `""`) are caught by Part 1/2's filter and, if nothing survives, dropped by Part 4. The `other` Edelweiss/Bharat-Bond rows are recovered intact (none trip a trigger), so they stop being mis-scored once the objective candidates are removed from the pool.
